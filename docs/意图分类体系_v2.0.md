# 招商银行智能客服意图分类体系 v2.0

> 基于CCCS国标 + 招行95555真实运营场景设计
> 版本：v2.0 | 日期：2026-05-31

---

## 一、设计原则

1. **对齐国标**：参考《信息技术服务 智能客户服务国家标准》(CCCS) 的五维模型
2. **贴合业务**：基于招行95555真实客服运营数据
3. **可评测**：每个意图有明确的触发条件和预期回答
4. **层级清晰**：一级大类 → 二级子类 → 三级具体意图

## 二、意图分类体系（3层结构）

### 2.1 一级分类（6大类）

| 一级分类 | 编码 | 说明 | 占比估算 |
|---------|------|------|---------|
| 信息查询类 | INFO | 账户、账单、进度等查询 | 30% |
| 业务办理类 | BIZ | 转账、挂失、密码等操作 | 20% |
| 咨询投诉类 | CONSULT | 产品咨询、投诉建议、紧急求助 | 20% |
| 营销推广类 | SALES | 理财、贷款、活动推广 | 10% |
| 安全风控类 | SECURITY | 盗刷、反诈、账户冻结 | 10% |
| 系统交互类 | SYSTEM | 问候、致谢、无效输入 | 10% |

## 三、P0紧急转人工规则

### 3.1 绝对P0（立即转人工）
以下意图必须100%转人工，不允许AI拦截：

1. **human_service**：用户明确说"转人工"、"找客服"、"人工服务"
2. **complaint**：用户表达不满/投诉
3. **urgent_help**：用户表达紧急情况（资金损失、账户锁定等）
4. **anti_fraud**：诈骗举报
5. **theft_report**：盗刷反馈
6. **freeze_request**：账户冻结申请
7. **security_event**：其他安全事件

### 3.2 条件P0（满足条件转人工）
- 用户表达情绪激动（愤怒、不满）
- 同一问题连续3次未解决
- 用户明确拒绝AI解决方案

## 四、风险提示规则

### 4.1 必须提示的场景
| 场景 | 提示内容 | 法规依据 |
|-----|---------|---------|
| 理财产品咨询 | "理财非存款，投资有风险" | 《证券期货投资者适当性管理办法》 |
| 贷款产品咨询 | "贷款需谨慎，请理性评估还款能力" | 《商业银行信用卡业务监督管理办法》 |
| 转账操作 | "请核实收款人信息，转账后无法撤销" | 《关于加强支付结算管理防范电信网络新型违法犯罪通知》 |
| 信用卡取现 | "信用卡取现按日计息，费率较高" | 《信用卡业务管理办法》 |

### 4.2 提示触发机制
- 意图识别为理财/贷款类产品咨询时，自动附加风险提示
- 转账金额>1万时，追加防欺诈提示
- 检测到敏感词（如"转账"、"汇款"）时，触发防骗提示

## 八、实施要点

1. **意图识别优先P0**：风控类意图识别置信度低时，倾向转人工
2. **情绪检测辅助**：检测到负面情绪时提升转人工概率
3. **多轮上下文**：复杂问题需保持对话上下文
4. **人机协同**：AI处理不了的立即转人工，不做无谓尝试
