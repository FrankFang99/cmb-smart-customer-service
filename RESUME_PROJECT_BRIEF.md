# 【分行 AI 智能客服业务探索与评测体系搭建项目】—— 500 字精简版 v3

**项目定位**：**分行 AI 智能客服业务探索与评测体系搭建**——**对公 + 零售一起搞，核心是"地方特色业务"**（分行有地方政策 / 地方活动 / 地方核心企业 / 地方文化，**总行云端大模型做不了**）。

**总分合作（A 模式，业界最前沿混合架构）**：**总行做"生成层"（借总行通义千问 RAG + 全行服务一家业务协同）+ 分行做"判断层 + 地方特色层"（BERT + L0 词典 + 565 条地方特色知识库 + 客户经理协同）**。**业界 100% 共识**——11 个真实案例全部是 A 模式（招行呼和浩特 / 佛山 / 深圳 / 日照、工行 1+X MoA、建行大模型+小模型、上海银行大模型+微模型等）。

**AI 边界（PM 价值观）**：**"AI 不是为了让 AI 接管对公 + 零售业务，是为了把客户经理从'通用产品答疑'里解放出来，专注'地方特色业务'"**——AI 该答地方特色 + 高频基础（薪福通/招贷 App/闪电贷/L0 红线），不该答应收账款贴现/科创贷授信/跨境融资方案等公司经营隐私+客户经理单独报价的复杂问题。

**项目结果（v3.5.5 → v3.6.0，真实数据）**：

| 指标 | baseline | v3.6.0 | 提升 |
|---|---|---|---|
| 意图准确率 | 61.88% | 67.04% | **+5.16pp** |
| P0 红线 Recall | 78.9% | 76.01% | 守住 |
| 外部 LLM 调用率 | 38% | **10.6%** | **-27.4pp**（89% 不依赖云端）|
| 自动化评测效率 | 96h/轮 | 14s/轮 | **+400%** |
| 业务组 sales（招贷 App/闪电贷/政采贷/地方特色）| 24.67% | 72.86% | **+48.19pp** |
| 业务组 cons（薪福通/对公账户/地方特色）| 50.15% | 57.01% | +6.86pp |
| **业务组 sys ⚠️** | 60.57% | **35.34%** | -25.23pp（cascade 误判，待修）|
