# 【分行 AI 智能客服业务探索与评测体系搭建项目】—— 500 字精简版 v4

> **简历范式**：按"项目背景 / 核心工作 / 项目结果"分块陈述（对标 AI 产品运营标准简历范式）

## 项目背景

分行侧普遍两个真问题：① **总行云端大模型做不了"地方特色业务"**（地方政策 / 地方活动 / 地方核心企业 / 地方文化）；② **分行没有完整客服团队**，对公 / 零售客户经理每天接 30-50 个重复答疑电话（薪福通怎么开通？招贷 App 怎么用？闪电贷利率多少？），**根本没时间跑客户、做尽调、做方案**。本项目目标：搭建分行 AI 智能客服业务体系（**对公 + 零售双轨**），让 AI 接管"通用产品高频问题 + 地方特色业务"双轨，把客户经理从重复答疑里解放出来，专注地方特色业务。

## 核心工作

1. **总分合作模式设计（A 模式，业界最前沿混合架构）**：调研招行 4 大分行（呼和浩特 / 佛山 / 深圳 / 日照）+ 工行 1+X MoA / 建行大模型+小模型 / 上海银行大模型+微模型 **11 个真实案例**，**业界 100% 共识**——总行做"生成层"（借总行通义千问 RAG + 全行服务一家业务协同），分行做"判断层 + 地方特色层"（BERT + L0 词典 + 565 条地方特色知识库 + 客户经理协同）。

2. **AI 边界与 L0/L1/L2/L3 产品分层**：AI 该答"**地方特色业务**（对公：制造业供应链 / 公积金 / 绿色金融 / 科技创新企业；零售：本地化营销 / 反诈 / 私钻 / 具身智能）+ **高频基础**（薪福通 / 招贷 App / 闪电贷 / L0 红线）"；AI 不该答"**公司经营隐私 + 客户经理单独报价 / 方案的复杂问题**"（应收账款贴现 / 科创贷授信 / 跨境融资方案等）。**这是 PM 坚持的 AI 边界**。

3. **Cascade L1→L2 BERT→L3 LLM 门控 + 5 件套防过拟合**：6 轮版本迭代（v3.1→v3.6.0），独立设计 7750 条黄金评测集（train/holdout 拆分，seed 42 可复现），自动化评测后台让 AI 自动给 AI 客服回复打分（评测效率 +400%）。**BERT 5 件套防过拟合**（早停 + 冻结 + weight_decay + dropout + label_smoothing）+ 0.85 置信度门控 + LLM 兜底。

4. **评测体系与 PM 诚实面对业务**：覆盖 4 维度（会不会胡说 / 答得对不对 / 答得全不全 / 有没有答非所问），**诚实修正统计噪声**（600 样本下 +5.17pp 修正为 +0.93pp），面试时**主动讲 sys 业务组下滑 25.23pp 的瑕疵**——体现 PM 诚实面对业务的能力。
