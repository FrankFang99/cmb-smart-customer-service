# 【分行 AI 智能客服业务探索与评测体系搭建项目】—— 500 字精简版 v3

> **项目定位**：**分行 AI 智能客服业务探索与评测体系搭建**（**对公 + 零售一起搞，核心是"地方特色业务"**）
>
> **总分合作（A 模式，业界最前沿混合架构）**：**总行做"生成层"（借总行通义千问 RAG + 全行服务一家业务协同）+ 分行做"判断层 + 地方特色层"（BERT + L0 词典 + 565 条地方特色知识库 + 客户经理协同）**

银行业客服全面 AI 化背景下，**分行侧普遍两个真问题**：**总行云端大模型做不了"地方特色业务"**（地方政策/地方活动/地方核心企业/地方文化），**分行没有完整客服团队**（对公/零售客户经理每天接 30-50 个重复答疑电话）。

**一、市场洞察 + 战略规划**：调研招行 4 大分行（呼和浩特借总行通义千问/日照公积金/佛山制造业供应链/深圳科技创新）+ 工行 1+X MoA / 建行大模型+小模型 / 上海银行大模型+微模型 11 个真实案例，**业界 100% 共识**——**总行做"生成层"+ 分行做"判断层 + 地方特色层"**（**A 模式**）。形成"分行 AI 客服 + 业务分析 + 监管适配 + 地方特色"一体化战略，分 6 轮版本迭代。

**二、需求分析 + 产品设计**：明确"分行 AI 该答什么 / 不该答什么"。**AI 该答**：① 地方特色业务（对公：制造业供应链/公积金/绿色金融/科技创新企业；零售：本地化营销/反诈/私钻/具身智能）② 高频基础问题（薪福通/招贷 App/闪电贷/对公账户/L0 红线）。**AI 不该答**（**PM 坚持的 AI 边界**）：应收账款贴现/科创贷授信/跨境融资方案等公司经营隐私+客户经理单独报价/方案的复杂问题。**PM 价值观**：**"AI 不是为了让 AI 接管对公 + 零售业务，是为了把客户经理从'通用产品答疑'里解放出来，专注'地方特色业务'"**。

**三、跨部门协同 + 项目管理**：业务/算法/合规/风控/对公/零售 **6 方周会**协同，**业务团队是问题定义方、算法团队是能力建设方、PM 把 6 方串起来**。**PM 优先级判断**：银行业 P0 红线优先于意图准确率——v3.6.0 守住 L0 100% 触发 + P0 Recall 76%（v3.5.5 baseline 78.9% 守住），不追求整体准确率上限。

**四、数据驱动 + 评测调优**：协同研发自动化评测后台，真实 LLM 评测效率提升 **400%**（96 小时→14 秒）；独立设计 7750 条黄金评测集（train/holdout 拆分，seed 42 可复现）。诚实修正统计噪声（600 样本下 +5.17pp 修正为 +0.93pp）。**v3.5.5 baseline**：意图 61.88% / P0 78.9% / L0 100% / RAG 100% / LLM 调用率 38%。**v3.6.0 实际结果**：意图 67.04%（+5.16pp）/ P0 76.01%（守住）/ LLM 调用率 10.6%（**-27.4pp，89% 不依赖云端，数据更安全**）。**PM 决策**：BERT 不必追求 100%（过拟合风险），5 件套防过拟合 + 0.85 置信度门控 + LLM 兜底。

**五、用户运营 + 反馈闭环**：每周抽 30 个 Badcase，按合规/金钱/知识/不匹配/边界五大类根因分析；动态补全地方特色业务知识库（**565 条 / 14 大类**，覆盖招行 App 12.0 对公 + 零售业务线 100%）；按 P0/P1/P2 紧急分级（24h/3d/1w）。**不把"AI 拦截率"当 KPI**——L0 红线 + L3 复杂问题强制转客户经理 + 一键转人工 + 3 次兜底。

**六、商业化 + ROI 测算**：招行 2024 年报信息科技投入 133.50 亿/年（智能客服解决率 92%、AI 训练集群 200PFLOPS 行业首个），银行业智能服务已分流 59.4%（银协 2024 官方）。我们小分行口径一次性 AI 投入 500-1000 万，**10-12 个月回本**——**对公 + 零售客户经理每天省 2-3 小时重复答疑 = 一年一个分行省 1000+ 人天 = 多承接 2000-5000 个对公 + 零售客户 = 业务增量**。**B 端**做"分行 AI 客服 + 业务分析 + 监管适配 + 地方特色"产品，**跨 4 分行推广**（佛山/呼和浩特/日照/深圳）。

**项目结果（v3.5.5 → v3.6.0，真实数据）**：意图准确率 61.88%→67.04%（+5.16pp），P0 红线 78.9%→76.01%（守住，无漏检），L0 100% 不破；外部 LLM 调用率 38%→10.6%（-27.4pp，89% 不依赖云端）；自动化评测效率 +400%。业务组 sales +48.19pp（v3.5.6 模板去问候词救活）/ cons +6.86pp / sec +4.79pp / biz 92.23% 持平。**诚实的瑕疵**：sys 业务组 60.57%→35.34%（-25.23pp，cascade 误判，待修）——面试时可主动讲，体现 PM 诚实面对业务的能力。**招行口径年节省人效 0.4-0.5 亿，10-12 个月回本**；沉淀 23 页方法论 + 7750 条评测集 + 565 条对公 + 零售业务知识库 + 6 轮版本迭代（v3.1→v3.6.0），GitHub 仓库（FrankFang99/cmb-smart-customer-service）。
