## 3. P1 误伤归因（93 条）

**根因**：Round 1 加的 freeze patterns 太宽。

典型误伤：
- "为什么我的卡被冻了?" → P1 询问 → 误判 P0 转人工
- "卡不能用了, 怎么解冻" → P1 询问 → 误判 P0 转人工
- "卡冻结了?" → P1 询问 → 误判 P0 转人工
- "大额存单" / "买个理财" → P1 业务 → 被 AML patterns 误伤

**Loop Engineering 反馈循环**：这些误伤会成为 **Round 2 patch 的输入**（缩 patterns + 加上下文约束）。

---

## 4. Round 2 计划

**触发条件**: Round 1 P1 误伤 93 条，需要 Round 2 缩 patterns。

**Round 2 patch 策略**：
1. **缩 freeze patterns**：删除 "卡被冻/账户被冻/卡冻结" 等询问意图 pattern
   - 只保留 "我要冻结/我要锁卡/冻结我的卡" 等明确执行意图
   - 加上下文约束：必须含 "要/帮我/请帮我" 等执行动词
2. **缩 AML patterns**：删除 "买理财/大额存单" 边缘 case
   - "让我转 + 对公账户" → 只保留 "让我转 + 数字 + 万/到对公" 强信号
3. **跑 D v3.2 + D v3.3 双验证**：
   - D v3.2 P1 准确恢复到 ≥72% (v3.10.1 水平)
   - D v3.3 P0 召回不破 100%

## 5. Loop Engineering 闭环纪律

### 5.1 方法论沉淀

| 步骤 | 动作 | 验证 |
|---|---|---|
| 1. 扩样本 | 多源真实 query (不上 LLM 改写) | D v3.3 = 206 条 |
| 2. 跑基线 | 在新样本上跑当前线上 patch | v3.10.1 = 96.61% (暴露过拟合) |
| 3. 写 patch | 只追加 generalizable patterns，不删旧的 | v3.11.0 Round 1 |
| 4. 双验证 | D 新样本 + D 老样本同时跑 | D v3.3: 100%, D v3.2: 不破 |
| 5. 沉淀负例 | 把误伤 query 固化进 negative_examples | 15 → Round 2 用 |
| 6. 下一轮 | Round 2 缩 patterns | 待跑 |

### 5.2 PM 视角

- **不抄权重**：v3.11.0 patch 不是 v3.10.1 patterns 加权，而是**叠加新 patterns**
- **诚实标注**：D v3.2 上 P1 准确 -0.9pp 不是 REGRESSION（minor），但要标出来让用户决定
- **真实样本 vs paraphrasing**：D v3.3 全部来自真实源，不靠 LLM 改写，避免套娃
- **过拟合暴露**：D v3.2 上 99.77% 是训练子集表现，D v3.3 上 96.61% 才是真实泛化

## 7. Round 1 结论

**Round 1 成功**（带瑕疵）：
- ✅ 扩充真正新样本（206 条，来自 4 个真实源）
- ✅ 重训不是加权（叠加 patterns，不删旧的）
- ✅ 沉淀负例库（15 条 P1 边界）
- ✅ Loop Engineering 闭环跑通（基线 → patch → 双验证 → 负例 → 下一轮）

- ✅ D v3.2 上 P0 不破（100%）
- ⚠️ P1 误伤 93 条（freeze patterns 太宽）→ Round 2 修

**PM 决策**：Round 1 patch 可以**作为 v3.11.0 候选**。Round 2 在不破 Round 1 P0 召回的前提下，缩 freeze / AML patterns 把 P1 准确恢复到 ≥72%。
