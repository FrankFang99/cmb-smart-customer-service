## 2. Round 2 策略

**核心**: 不是简单删除误伤 patterns, 而是**重新设计语义约束**

### freeze 收紧
**保留** (执行意图 + 紧急态):
- "我要冻结/帮我冻结/请帮我冻结" (执行动词)
- "账户状态异常/不对/有问题" (紧急态描述)
- "账户被锁/卡被锁" (报告锁定状态)

**删除** (Round 1 误伤源):
- "为什么我的卡被冻了?" (询问意图)
- "卡不能用了, 怎么解冻" (询问解冻)
- "卡冻结了?" (描述状态但实为询问)

### AML 收紧
**保留** (强 P0 信号):
- "让我转 X 万" (指示语 + 数字)
- "X 万转对公/X 万投资款/X 万货款" (数字 + 幌子)

**删除**:
- "投资款/货款" 单独词 (太宽)
- "对公账户投资" 不带数字

### 累积负例库
- Round 1 baseline: 18 条 (手写)
- Round 1 regression: 30 条 P1 误伤
- **总计: 66 条 P1 边界** (`negative_candidates_v311_r2.jsonl`)

## 4. Round 2 P1 误伤归因 (89 条)

### 4.1 freeze 误伤 (62 条)
R2 收紧但 freeze 子类 D v3.2 中"询问意图" query 太多:
- "为什么我的卡被冻了?" / "卡冻结了?" / "为啥账户被冻" / "账户冻结, 怎么解"
- 用户**描述状态但实际是询问意图**, R2 收紧没完全到位
- 根因: 中文"被冻/被锁/状态异常"可以是 **P0 (报告紧急)** 也可是 **P1 (询问)**, 需上下文

### 4.2 AML/其他误伤 (11 条)
- "大额存单" / "买理财" / "影票 9 块 9" / "定投" / "开指纹登录改默认卡"
- 这些是 P1 业务/产品查询, 触发 freeze 或 AML patterns

### 4.3 Round 2 新增误伤 (16 条, R1 没误伤)
- "我密码输错被锁了" / "我登录密码忘了被锁了" → 含 "被锁" R2 命中 freeze
- "我卡不能用了" → "不能用了" 在 R2 删了, 但 D v3.2 期望 P1, R2 也命中
- "我卡被冻结了想解冻" → "被冻结了" 命中
- "美元汇率多少我要汇 5 万美元给美国" → "汇 5 万" 触发 AML
- "请问账户状态不对" / "请问账户状态异常" → R2 保留了紧急态, 但触发了误伤
- "我的卡被锁了" → 描述状态, R1 删了 R1 没误伤, R2 又保留了 → 误伤回归

## 5. Loop Engineering 闭环复盘

| 轮次 | 动作 | 关键学习 |
|---|---|---|
| **v3.10.1** | baseline | 在 D v3.2 (训练子集) 上 99.77%, 但 D v3.3 (真实未见) 上 96.61% |
| **Round 1** | 扩 patterns 修漏检 | D v3.3 = 100%, 但 P1 误伤 93 (freeze + AML 太宽) |
| **Round 2** | 收紧 patterns | D v3.2 P1 恢复到 73.19% (反超 v3.10.1), 但 D v3.3 漏 1 条 + R2 新增 16 条误伤 |

### 5.1 真实发现

**Round 1 vs Round 2 都不是"完美"**:
- R1: P1 -0.90pp, 误伤 93
- R2: P1 +0.30pp, 误伤 89 (但漏 1 条)

**P1 准确差距** (72.89% vs 73.19%) < 1pp, 在**统计噪声范围内**——本质是 D v3.2 里 freeze/AML 边界 query 太多, 单靠 patterns 收紧无法彻底解决。

**Loop Engineering 教训**:
- patch 不能同时优化 P0 (漏检) 和 P1 (误伤) —— 中文"被冻/被锁/异常"是**天然 P0/P1 边界模糊带**
- **真正解法不是更多 patterns, 而是上下文消歧 (BERT 微调或 LLM 兜底)**

## 6. 最终 Round 1+2 合并结论

**v3.11.0 final 推荐 = Round 1 patch**:
- D v3.2 P0 = 100% ✓
- D v3.3 P0 = 100% ✓ (没漏)
- D v3.2 P1 = 71.99% (minor -0.90pp, 误伤 93)

**理由**: Round 2 在 D v3.3 上**漏 1 条 freeze** ("想问一下我的卡怎么锁住了"), 而 Round 1 没漏。

**Round 2 价值**:
- 验证了"收紧 P1 误伤"的思路可行 (89 < 93)
- 累积了 66 条 P1 边界负例库 (regression check 资产)
- 沉淀了"P0/P1 边界模糊带"的认知 —— 后续 patch 必须有上下文消歧
