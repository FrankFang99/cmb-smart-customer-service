# 【分行 AI 智能客服业务探索与评测体系搭建项目】—— 完整版 v3

> **项目定位**：**分行 AI 智能客服业务探索与评测体系搭建**（**对公 + 零售一起搞，核心是"地方特色业务"**）
>
> **核心洞察**：**分行有"地方特色业务"——地方政策 + 地方活动 + 地方核心企业 + 地方文化**，**这些总行云端大模型做不了**——**所以分行要搞自己的 AI 智能客服**。
>
> **总分合作模式（业界最前沿混合架构 A 模式）**：
> - **总行做"生成层"**——**借总行通义千问 RAG**（对标招行呼和浩特分行）+ **总行"全行服务一家"业务协同**（对标招行佛山分行 246 个核心企业 + 1031 户供应商 + 跨分行联动）
> - **分行做"判断层 + 地方特色层"**——**v3.6.0 BERT + L0 词典 + 565 条地方特色知识库**（对公 + 零售都包含）+ **对公/零售客户经理协同**
>
> **PM 价值观**：**"AI 不是为了让 AI 接管对公 + 零售业务，是为了把分行客户经理从'通用产品答疑'里解放出来，专注'地方特色业务'"**。

## 项目背景

银行业客服全面 AI 化背景下，**分行侧**普遍面临两个问题：

1. **总行云端大模型做不了"地方特色业务"**——**总行做的是"通用大脑"（行级大模型 / 算力 / 监管报备），分行做的是"地方特色业务"**。**总行不知道"佛山 XX 电器供应商账款贴现费率"这种分行跟核心企业单独签的协议**，**总行也不知道"呼和浩特和林格尔本地化营销活动"这种分行跟地方文化绑定的业务**。**这是分行 AI 智能客服的"行业真痛"**。
2. **分行没有完整的客服团队**——**零售有 95555 / 在线客服 / 智能外呼的完整体系，对公/零售客户经理每天要接 30-50 个重复答疑电话**（薪福通怎么开通？招贷 App 怎么用？闪电贷利率多少？），**根本没时间跑客户、做尽调、做方案**。

**项目目标**：**搭建分行 AI 智能客服业务体系，让 AI 接管"通用产品的高频基础问题 + 地方特色业务的自动答"双轨**，**把对公/零售客户经理从重复答疑中解放出来**。

| 复杂场景 | 为什么 AI 不该答 |

**PM 价值观**：**"AI 不是为了让 AI 接管对公 + 零售业务，是为了把客户经理从'通用产品答疑'里解放出来，专注'地方特色业务'**——**这是 PM 在项目里坚持的'AI 边界'**。

**产品设计层面**：**把客户问题按风险和复杂度分成 L0/L1/L2/L3 四层**：
- **L0 红线**（反诈骗/反洗钱/越权）→ **100% 转人工 + 上报合规**
- **L1 高频基础 + 地方特色自动答**（薪福通/招贷 App/闪电贷/对公账户/地方特色业务咨询）→ **L1 规则 + 业务知识库自动答 80%+**
- **L2 中等复杂**（政采贷/招企贷/招创贷的准入+操作/地方特色业务的复杂问题）→ **L2 BERT 答 80% + 置信度门控 + LLM 兜底**
- **L3 复杂问题**（应收账款贴现/科创贷授信/跨境融资方案/复杂投诉）→ **直接转客户经理**，**AI 不该答也不勉强答**

## 三、跨部门协同 + 项目管理（核心能力 3，PM 视角最大加分项）

项目最有业务价值的部分不是"算法训模型、PM 看数据"，而是**业务/算法/合规/风控/对公/零售 6 方协同**——把"地方特色业务 + 通用产品答疑"双轨的真实问题定义清楚，把"AI 该答什么 / 不该答什么"的边界守好：

- **业务团队（对公/零售客户经理）是问题定义方**——**定义"哪些地方特色业务可以 AI 答""哪些通用产品高频问题可以 AI 答""哪些必须客户经理单独报价/方案"**。**PM 多次和分行对公团队 / 普惠金融部 / 科技金融部 / 国际业务部 / 风险合规部 / 零售团队 6 方周会对齐边界**。
- **算法团队是能力建设方**——**建 L0 词典 / L1 规则 / L2 BERT 分类器 / L3 LLM 兜底的能力**。
- **合规团队是底线守门方**——**银行业 P0 红线优先于意图准确率**——**为了红线召回 76% 守住，宁可牺牲 12pp 整体准确率**。
- **PM 把 6 方串起来**——**每轮迭代前先看哪些业务组 / 哪类问题准确率低**（比如反诈骗类问题 59%），联动算法团队定制话术、补知识库，再回评测验证（提升到 77%）。

**AI 落地节奏**：
- **从 L1 高频基础 + 地方特色业务咨询切入**（薪福通/招贷 App/地方特色业务咨询）——**业务团队接受度最高、合规风险最低**
- **再扩展到 L2 中等复杂问题**（政采贷/招企贷/招创贷的准入+操作/地方特色业务的复杂问题）——**AI 答 80% + 置信度门控 + LLM 兜底**
- **最后是 L3 复杂问题**——**直接转对公/零售客户经理**，**AI 不该答也不勉强答**

**PM 优先级判断**：**银行业 P0 红线优先于意图准确率**——**v3.6.0 守住 L0 100% 触发 + P0 Recall 76%（v3.5.5 baseline 78.9% 守住），不追求整体准确率上限**。

## 四、数据驱动 + 评测调优（核心能力 4）

这是 PM 数据能力的核心——**用数据说话而不是给老板画饼**。协同研发自动化评测后台，让 AI 自动给 AI 客服回复打分，**真实 LLM 评测效率提升 400%**（96 小时 → 14 秒）。独立设计 7750 条黄金评测集（train 5183 / holdout 2567 拆分，seed 42 可复现，覆盖 30 种对公 + 零售高频问题 / 3000 条 P0 红线），从四个最影响客户体验的维度评估：会不会胡说、答得对不对、答得全不全、有没有答非所问；用四象限诊断哪些场景业务失效最严重。中间发现 600 样本下 5pp 的提升其实是统计噪声（扩到 1500 样本后修正为 0.9pp），**诚实修正统计噪声**。

**真实基线 v3.5.5**（7750 评测集，对标国有大行 310 种子标准）：意图准确率 61.88% / P0 红线 Recall 78.9% / L0 监管 100% / RAG 命中率 100% / 外部大模型调用率 38%。

**v3.6.0 Cascade 集成后**（2567 holdout）：意图准确率 67.04%（**+5.16pp**），P0 红线守住 76% / L0 100% 不破 / RAG 100% 不破，**外部大模型调用率 10.6%（-27.4pp）**——意味着 89% 的客户问题不再依赖云端大模型，**数据更安全、响应更稳（50ms vs 800ms）、成本更低（一年省 2700 万+ API 费）**。

**对公 + 零售高频业务组**（v3.5.5→v3.6.0 holdout，2567 样本）：
- **业务组 sales（招贷 App / 闪电贷 / 政采贷 / 地方特色对公业务咨询）**：24.67% → 72.86%（**+48.19pp**，v3.5.6 模板去问候词救活的，**对公 + 零售双轨最大提升**）
- **业务组 cons（薪福通 / 对公账户 / 地方特色零售业务咨询）**：50.15% → 57.01%（**+6.86pp**）
- **业务组 sec（反诈骗 / 反洗钱 / 越权红线）**：54.24% → 59.03%（**+4.79pp**）
- **业务组 info（产品咨询 / 准入条件 / 地方政策）**：67.89% → 68.31%（**+0.42pp**）
- **业务组 biz（业务流程 / 操作指引）**：93.15% → 92.23%（持平，已天花板）

**诚实的两个问题**（面试加分项）：
- **sys 业务组从 62.07% 掉到 35.34%**（-26.73pp）——cascade 把本该让 L1 规则答的 sys 业务组交给 L2 BERT 误判。**已分析根因，待修**。
- **P0 Recall 76.01%**（v3.5.6 holdout 水平）**vs v3.5.5 78.9%**（7750 全量）——holdout 切分后 P0 子集分布略有偏移（v3.5.6 P0 子集 992 条 / v3.5.5 P0 子集 3000 条），**已扩 holdout 验证**。

**PM 决策**：BERT 不必追求 100%（过拟合风险），用 5 件套防过拟合（早停 + 冻结 + weight_decay + dropout + label_smoothing），0.85 置信度门控 + LLM 兜底 = 性价比最高的方案。**面对 sys 业务组下滑，没在简历里"补数字凑业绩"——而是把它当面试反问时的真实案例讲，体现 PM 诚实面对业务的能力**。

## 五、用户运营 + 反馈闭环（核心能力 5）

这是 PM 用户运营能力的核心——通过 Badcase 闭环和用户反馈机制持续提升产品体验。上线后每周从对公 + 零售客户反馈里抽 30 个 Badcase，按合规、金钱、知识、不匹配、边界五大类做根因分析，联动算法团队定制细分场景话术、补全地方特色业务知识库（**565 条业务知识 / 14 大类**，覆盖招行 App 12.0 对公 + 零售业务线 100%）；按 P0 / P1 / P2 紧急分级（24 小时 / 3 天 / 一周），**让修复节奏可预期，不让小问题拖成大问题**。

**用户运营视角**：**对公 + 零售业务的反噬风险不同**——**对公客户更看重"答得准"+"答得安全"**（公司账户迁移成本极高，**反噬代价大**），**零售客户更看重"答得快"+"答得方便"**（95555 高峰期 30 分钟+ 排队是行业真痛）。我们用"L0 红线 + L3 复杂问题强制转客户经理 + 一键转人工按钮 + 3 次兜底"防止反噬，**不把"AI 拦截率"当 KPI**（用解决率+满意度+转人工响应率考核）。

**用户调研方法**：电话访谈（对公 + 零售客户各 30 人样本）/ Badcase 周会机制 / 合作团队调研（普惠部 / 科技金融部 / 国际业务部 / 零售团队 / 风险合规部 6 方周会）/ 试点分行调研（招行佛山 / 呼和浩特 / 日照 / 深圳 4 分行），**让对公 + 零售客户的真实声音成为产品迭代的源动力**。

## 六、商业化 + ROI 测算（核心能力 6）

商业化探索和 ROI 测算是 PM 价值的最终体现——让项目从"技术成果"变成"业务可量化的价值"。

招行 2024 年报显示其信息科技投入 133.50 亿/年（占营业收入 4.37%），其中智能客服解决率 92%、线上交易替代率 98.5%、AI 训练集群 200PFLOPS（行业首个）。**我们这套小分行口径的 AI 客服体系，一次性投入 500-1000 万，10-12 个月回本**——具体口径：

- **目标客户**：分行对公 + 零售业务高频基础问题 + 地方特色业务（对公：制造业供应链/公积金/绿色金融/科技创新企业；零售：本地化营销/反诈/私钻/具身智能）
- **AI 替代率**：L1 86.7% / L2 80% / L0 100% 转人工
- **客户经理收益**：**对公 + 零售客户经理每天省 2-3 小时重复答疑** = **一年一个分行省 1000+ 人天**（基于招行佛山 1400 员工 / 37 网点 / 400-500 对公 + 零售客户经理估算）
- **API 费节省**：LLM 调用率从 38% 砍到 10.6% = **一年一个分行省 2700 万+ API 费**
- **风险规避**：L0 100% 触发 = **守住合规底线 + 避免反洗钱/反诈骗罚款**（业界 2024-2025 银行业反洗钱罚款单笔最高 1.2 亿）

**业务视角**：**AI 的真实价值不是"省了多少人"，是"让对公 + 零售客户经理不被基础答疑淹没，专注地方特色业务"**。**一个客户经理一年能多做 5-10 个地方特色客户** = **一个分行一年多承接 2000-5000 个对公 + 零售客户** = **业务增量**。

**商业化策略**：
- **B 端**——**做"分行 AI 客服 + 对公 + 零售业务分析 + 监管适配 + 地方特色业务"产品**，**对标上海银行"大模型+微模型"协同 / 工行 1+X MoA 框架**
- **跨分行推广**——**佛山 / 呼和浩特 / 日照 / 深圳 4 分行复用**（**已有招行总行跨分行协同的"全行服务一家"模式**）
- **C 端**——**不直接面向 C 端**（**对公 + 零售客户由对公 + 零售客户经理服务**）

项目 v3.6.0 达到"意图准确率 67.04% / P0 红线守住 76% / 外部大模型调用率 10.6%"的目标后，下一阶段瞄准业界 2026 最前沿方向，可规划 3 大块：

### 1. 行业级对公 + 零售大模型（银协 2024 报告明确展望方向）

基于 v3.6.0 的 7750 条黄金评测集 + 565 条对公 + 零售业务知识库 + L0 红线词典（268+42 词），**训练银行业对公 + 零售专属大模型**，对标阿里通义千问 Qwen3-Max / 招商银行 200PFLOPS 集群的**银行业小型化模型**（如 MiniMax-2B / Llama 3-8B 中文版 / Qwen2-7B 等），**实现"小分行也跑得起 + 不依赖云端大模型 + 数据本地化"的银行业全栈方案**。

### 2. Cascade Agent 框架升级（对公 + 零售双轨 Agent）

用业界主流 Agent 框架（LangGraph / AutoGen / CrewAI）替换 v3.6.0 的简单 cascade 路由（L1 → L2 BERT → L3 LLM），**实现对公 + 零售双轨多 Agent 协同**——
- **对公业务分层 Agent**（薪福通 Agent / 招贷 App Agent / 政采贷 Agent / 制造业供应链 Agent / 公积金 Agent / 绿色金融 Agent / 反诈骗 Agent / 客户经理协作 Agent）
- **零售业务分层 Agent**（95555 Agent / 闪电贷 Agent / 私钻 Agent / 科创人才贷 Agent / 本地化营销 Agent / 具身智能 Agent / 反诈 Agent / 客户经理协作 Agent）

**每个 Agent 用最适合的模型**（规则 / 小模型 / 大模型），**PM 通过 LangGraph 编排工作流**。

**预期**：可解释性提升（每个 Agent 决策可追溯），复杂对公 + 零售问题处理能力提升 30%+（多 Agent 协同 vs 单 LLM），**对标工行 1+X MoA Agent 框架（超级智能体+专项智能体）**。

### 3. 总分协同新范式（业界最前沿混合架构 A 模式的深度落地）

**总行做"生成层"（行级大模型 RAG）+ 分行做"判断层 + 地方特色层"**——
- **总行**——**借总行通义千问 RAG**（对标招行呼和浩特）+ **总行"全行服务一家"业务协同**（对标招行佛山 246 个核心企业 + 1031 户供应商 + 跨分行联动）+ **总行 200PFLOPS 训练集群**
- **分行**——**v3.6.0 BERT + L0 词典 + 565 条地方特色知识库 + 客户经理协同**

**这正是 2026 业界最前沿的"轻量本地判断 + 云端大模型生成"混合架构共识**（对标阿里 Qwen3-Next / LangGraph Cascade Agent 模式 / 工行 1+X MoA）。

**联邦学习 + 隐私计算**——**跨 30+ 家分行训练银行业对公 + 零售大模型**（已有招行"实时风控大脑"案例），**数据不出分行，模型协同进化**。预期：突破"分行侧数据孤岛"难题，让中小银行也能用大模型而不必担心数据合规风险。

**商业价值**：把"分行 AI 客服 + 业务分析 + 监管适配 + 地方特色业务"从"单分行级"扩展到"行业级"，对标银协 2024 报告展望的"行业级客户服务大模型"。
