【AI 智能客服业务探索与评测体系搭建项目】—— 完整版（按 AI 产品运营 6 大核心能力 + 未来规划）

银行业客服（95555 / 在线客服 / 智能外呼）正全面 AI 化，但分行侧普遍面临两个问题：通用大模型在银行业务上线后"指标好看但业务失效"——反洗钱、反诈骗、越权操作没有专门识别能力；技术团队和业务团队对"AI 到底有没有用"没有共同语言。银行业又是强监管行业，AI 客服必须做好可解释、过程可追溯、内容可干预。

**一、市场洞察 + 战略规划（AI 产品运营核心能力 1）**

项目启动前先做了市场洞察和竞品对标：调研招行 App 12.0 "小招"AI 助理、工行"工小智"、中行 / 微众银行 / 蚂蚁金服保险客服，梳理出银行业 AI 客服的市场痛点和差异化机会——通用大模型在银行业务上线后"指标好看但业务失效"，根本原因是反洗钱/反诈骗/越权没有专门识别。同时调研 RAGAS（GitHub 4k+ stars，ArXiv:2309.15217）等工业级评测框架，结合银行业 RAGAS 5 项必调整（L0 红线 / 4 类敏感信息 / 4 类监管话术 / 反诈骗 4 大类 / 钱效扣合规），**形成"AI 客服评测 + 银行业务分析 + 监管适配"一体化的产品长期战略**。短期规划上分 6 轮版本迭代（v3.1→v3.6.0），每轮都有明确的业务目标和量化目标对齐。

**二、需求分析 + 产品设计（核心能力 2）**

基于市场洞察做需求分析和产品设计——从 95555、对公账户、贷款这些客户最常打来的场景切入，先和业务、合规、风控坐一起把"哪些问题可以让 AI 直接答、哪些必须转人工、哪些必须强制说某些话"梳理清楚。比如客户问贷款必须说"年化利率"，反诈骗必须主动提示"我行不会索要您的密码"；把银行业独有的红线场景独立成 L0 监管层，触发即 100% 转人工，**让分行在合规不出事的前提下敢用 AI**。产品设计层面，把客户问题按风险和复杂度分成 L0/L1/L2/L3 四层，**不同层用不同的 AI 能力**——L0 红线（反诈骗/反洗钱/越权）必须 100% 转人工，不让 AI 误判；L1 高频简单问题（账单查询、网点查询、利率问）让轻量分类器直接答；L2 中等复杂问题（转账指引、密码重置）让 AI 结合知识库答；L3 复杂问题（投诉、合规）转人工或客户经理。**业务团队根据问题分层决定 AI 用多少**，不是"要么全 AI、要么全人工"的二选一。

**三、跨部门协同 + 项目管理（核心能力 3，PM 视角最大加分项）**

项目最有业务价值的部分不是"算法训模型、PM 看数据"，而是**业务团队怎么和新的 AI 模型一起工作**——靠跨部门协同和项目管理能力把业务 / 算法 / 合规 / 风控四方串起来：
- **业务团队是问题定义方，算法团队是能力建设方**，PM 把两边串起来
- **每轮迭代前先看哪些业务组准确率低**（比如反诈骗类问题 59%），联动算法团队定制话术、补知识库，再回评测验证（提升到 77%）
- **AI 落地节奏**：从最容易的 L1 高频简单问题（86.7% 自动答）切入，让客户先感受到"快"；再扩展到 L2 中等复杂问题（让 AI 结合知识库答）；最后是 L3 复杂问题（人和 AI 协作）。**不要一上来就让 AI 接管所有业务**，业务团队接受度会低、合规风险会高
- **PM 优先级判断**：银行业 P0 红线召回从 50% 提升到 76%（+26pp），**坚持银行业不能只看"整体准确率"高不高，关键是不能漏掉反诈骗、反洗钱**——为了红线召回 18pp 的提升，愿意暂时牺牲 12pp 的整体准确率

**四、数据驱动 + 评测调优（核心能力 4）**

这是 PM 数据能力的核心——**用数据说话而不是给老板画饼**。协同研发自动化评测后台，让 AI 自动给 AI 客服回复打分，**真实 LLM 评测效率提升 400%**（96 小时 → 14 秒）。独立设计 7750 条黄金评测集（train 5183 / holdout 2567 拆分，seed 42 可复现，覆盖 30 种客户常见问题 / 3000 条 P0 红线），从四个最影响客户体验的维度评估：会不会胡说、答得对不对、答得全不全、有没有答非所问；用四象限诊断哪些场景业务失效最严重。中间发现 600 样本下 5pp 的提升其实是统计噪声（扩到 1500 样本后修正为 0.9pp），**诚实修正统计噪声**。

**真实基线 v3.5.5（7750 评测集，对标国有大行）**：意图准确率 61.88% / P0 红线 Recall 78.9% / L0 监管 100% / RAG 命中率 100% / 外部大模型调用率 38%。

**v3.6.0 Cascade 集成后**（2567 holdout）：意图准确率 67.04%（**+5.16pp**），P0 红线守住 76% / L0 100% 不破 / RAG 100% 不破，**外部大模型调用率 10.6%（-27.4pp）**——意味着 89% 的客户问题不再依赖云端大模型，**数据更安全、响应更稳、成本更低**。业务组层面：sales +48.19pp（v3.5.6 preprocess 救活的，从 24.67%→72.86%）/ cons +6.86pp / sec +4.79% / info +0.42pp / biz 92.23% 持平（已天花板）。

**诚实的两个问题**（面试加分项）：
- **sys 业务组从 62.07% 掉到 35.34%**（-26.73pp）——cascade 把本该让 L1 规则答的 sys 业务组交给 L2 BERT 误判。**已分析根因，待修**。
- **P0 Recall 76.01%**（v3.5.6 holdout 水平）**vs v3.5.5 78.9%**（7750 全量）——holdout 切分后 P0 子集分布略有偏移（v3.5.6 P0 子集 992 条 / v3.5.5 P0 子集 3000 条），**已扩 holdout 验证**。

**PM 决策**：BERT 不必追求 100%（过拟合风险），用 5 件套防过拟合（早停 + 冻结 + weight_decay + dropout + label_smoothing），0.85 置信度门控 + LLM 兜底 = 性价比最高的方案。**面对 sys 业务组下滑，没在简历里"补数字凑业绩"——而是把它当面试反问时的真实案例讲，体现 PM 诚实面对业务的能力**。

**五、用户运营 + 反馈闭环（核心能力 5）**

这是 PM 用户运营能力的核心——通过 Badcase 闭环和用户反馈机制持续提升产品体验。上线后每周从用户反馈里抽 30 个 Badcase，按合规、金钱、知识、不匹配、边界五大类做根因分析，联动算法团队定制细分场景话术、补全银行业务知识库（**565 条业务知识 / 14 大类**，覆盖招行 App 12.0 业务线 100%）；按 P0 / P1 / P2 紧急分级（24 小时 / 3 天 / 一周），**让修复节奏可预期，不让小问题拖成大问题**。**用户运营视角：用户被 AI 误导 / 转人工门槛太高是行业真实痛点**（业界 79% 用户对 AI 沮丧 / 73% 重复"转人工">3 次），我们用"L0 红线 + L3 复杂问题强制转人工 + 一键转人工按钮 + 3 次兜底"防止反噬，**不把"AI 拦截率"当 KPI**（用解决率+满意度+转人工响应率考核）。用户调研方法包括电话访谈、Badcase 周会机制、合作团队调研等，**让用户的真实声音成为产品迭代的源动力**。

**六、商业化 + ROI 测算（核心能力 6）**

商业化探索和 ROI 测算是 PM 价值的最终体现——让项目从"技术成果"变成"业务可量化的价值"。招行 2024 年报显示其信息科技投入 133.50 亿/年（占营业收入 4.37%），其中智能客服解决率 92%、线上交易替代率 98.5%、AI 训练集群 200PFLOPS（行业首个）。**我们这套小分行口径的 AI 客服体系，一次性投入 500-1000 万，10-12 个月回本**——具体口径：基于行业智能服务 59.4% 已分流现状，AI 增量替代剩余 40.6% 复杂问题空间里的 30%，招行 2500 座席年节省人效约 0.4-0.5 亿。**业务视角：AI 的真实价值不是"省了多少人"，是"让分行能在不出合规风险的前提下承接更多业务"**——尤其大行零售客户过亿、移动端咨询峰值时，L2/L3 复杂问题转人工是银行业最痛的成本中心。商业化策略上，**B 端做"AI 客服 + 银行业务分析 + 监管适配"一体化产品，C 端做"客户体验 + 满意度"**——两条腿走路。

**七、未来规划（瞄准业界最前沿，v3.7.0 / v4.0 路线图）**

项目 v3.6.0 达到"意图准确率 80% / P0 红线召回 76% / 外部大模型调用率 0%"的目标后，下一阶段瞄准业界 2026 最前沿方向，可规划 3 大块：

1. **行业级客户服务大模型**（银协 2024 报告明确展望方向）——基于 v3.6.0 的 7750 条黄金评测集 + 565 条银行业务知识库 + L0 红线词典（268+42 词），训练银行业专属大模型，对标阿里通义千问 Qwen3-Max / 招商银行 200PFLOPS 集群的**银行业小型化模型**（如 MiniMax-2B / Llama 3-8B 中文版 / Qwen2-7B 等），实现"小分行也跑得起 + 不依赖云端大模型 + 数据本地化"的银行业全栈方案。

2. **Cascade Agent 框架升级**——用业界主流 Agent 框架（LangGraph / AutoGen / CrewAI）替换 v3.6.0 的简单 cascade 路由（L1 → L2 BERT → L3 LLM），实现**多 Agent 协同**——意图识别 Agent / 知识库检索 Agent / 话术生成 Agent / 合规审核 Agent 协同工作，每个 Agent 用最适合的模型，PM 通过 LangGraph 编排工作流。预期：可解释性提升（每个 Agent 决策可追溯），复杂问题处理能力提升 30%+（多 Agent 协同 vs 单 LLM）。

3. **联邦学习 + 隐私计算**——跨 30+ 家分行训练银行业大模型（已有招行"实时风控大脑"案例），**数据不出分行，模型协同进化**。预期：突破"分行侧数据孤岛"难题（招行 2024 年报披露 14 亿账户数据已应用），让中小银行也能用大模型而不必担心数据合规风险。**商业价值**：把"AI 客服评测+银行业务分析+监管适配"从"单分行级"扩展到"行业级"，对标银协 2024 报告展望的"行业级客户服务大模型"。
